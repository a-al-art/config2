# Визуализатор графа зависимостей Maven-пакетов

Данный инструмент позволяет анализировать и визуализировать зависимости Java-пакетов в формате Maven без использования готовых менеджеров пакетов или сторонних библиотек для получения зависимостей.

## Основные возможности

- **Автоматическое определение полных координат пакета** (groupId:artifactId:version) по имени артефакта (например, `commons-logging`).
- **Получение прямых зависимостей** (этап 2).
- **Построение полного транзитивного графа зависимостей** с помощью итеративного DFS (без рекурсии).
- **Визуализация зависимостей в виде ASCII-дерева**.
- **Фильтрация пакетов по подстроке** (исключаются из анализа и визуализации).
- **Обнаружение и корректная обработка циклических зависимостей**.
- **Поддержка тестового режима** с графом из JSON-файла (пакеты обозначаются заглавными латинскими буквами: A, B, C…).

## Этапы реализации

### Этап 1. Минимальный прототип с конфигурацией

- Реализован CLI-интерфейс с настраиваемыми параметрами через командную строку.
- Поддерживаемые параметры:
  - `--package` — имя анализируемого пакета.
  - `--repo` — URL репозитория Maven или путь к тестовому JSON-файлу.
  - `--test-mode` — режим работы (file или url).
  - `--ascii-tree` — включить вывод в виде ASCII-дерева.
  - `--filter` — исключить пакеты, содержащие указанную подстроку.
- При запуске выводятся все параметры в формате ключ-значение.
- Реализована обработка ошибок для всех параметров.

### Этап 2. Сбор данных

- Поддержка формата пакетов Maven.
- Автоматическое определение `groupId` и версии по `artifactId` (эвристика: `groupId == artifactId`).
- Извлечение прямых зависимостей из POM-файла по URL репозитория.
- Вывод списка прямых зависимостей.

### Этап 3. Основные операции

- Построение полного графа транзитивных зависимостей с помощью итеративного DFS (без рекурсии).
- Фильтрация пакетов на этапе построения графа.
- Обнаружение циклов и пометка их в дереве как `(.cycle.)`.
- Поддержка тестового режима с JSON-описанием графа.
- Рекурсивная визуализация построенного графа в виде ASCII-дерева.

## Примеры использования

- **Прямые зависимости пакета (этап 2):**
  ```bash
  python main.py --package commons-logging --repo https://repo1.maven.org/maven2 --test-mode url
    ```
  
- **Полное дерево зависимостей (этап 3):**
    ```bash
  python main.py --package commons-logging --repo https://repo1.maven.org/maven2 --test-mode url --ascii-tree
    ```
- **Тестовый режим с циклическим графом. Создайте файл graph.json со следующим содержимым:**
    ```json
    {
      "A": ["B", "C"],
      "B": ["D"],
      "C": ["D", "E"],
      "D": [],
      "E": ["A"]
    }
  ```
  
  ```bash
  python main.py --package A --repo graph.json --test-mode file --ascii-tree
  ```
## Для пакетов, где groupId != artifactId, требуется указывать полные координаты вручную
### Требования 
- Python 3.7+.
- Стандартная библиотека Python.
